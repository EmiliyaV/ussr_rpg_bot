# Красная развилка — USSR RPG Bot MVP

«Красная развилка» — интерактивная историческая ролевая игра в формате Telegram-бота. Игрок выбирает роль и проходит 20 ходов: с 1920 по 1939 год. К 1940 году игра подводит итог личной судьбы персонажа.

Игрок не меняет крупные исторические события. НЭП, образование СССР, индустриализация, коллективизация, Большой террор и кризис 1939 года остаются историческими рамками. Выборы влияют на личное положение: доверие власти, безопасность, влияние, ресурсы, отношение людей и накопленную биографию.

## Что есть

- Telegram-бот на aiogram 3.
- 3 роли: идейный коммунист, прагматик, скрытый противник советской власти.
- 20 игровых лет: 1920–1939.
- Исторический контекст, реальные факты года, ограничения игрока и варианты выбора.
- Скрытые показатели персонажа.
- Теги решений и ключевые события личной биографии.
- OutcomeResolver для критических годов и отложенных последствий.
- Ollama для художественных итогов года и финала.
- Fallback-режим без Ollama.
- SQLite для хранения состояния игры.

## Команды бота

- /start — начать новую игру.
- /status — показать текущее положение без чисел и технических метрик.
- /help — справка.
- /debug_status — отладочная команда со скрытыми числами. По умолчанию отключена. Включается только через DEBUG_COMMANDS_ENABLED=true.

## Переменные окружения

Создай .env из .env.example.

Linux/macOS:
cp .env.example .env

Windows PowerShell:
copy .env.example .env

Минимально нужно заполнить:
TELEGRAM_BOT_TOKEN=...

Ollama можно отключить:
OLLAMA_ENABLED=false

## Запуск

Создать виртуальное окружение:
python -m venv .venv

Windows PowerShell:
.\.venv\Scripts\Activate.ps1

Linux/macOS:
source .venv/bin/activate

Установить зависимости:
pip install -r requirements.txt

Проверки:
python scripts/check_history_contexts.py
python scripts/check_project_steps.py
python scripts/check_final_polish.py

Запуск:
python -m app.main

## Ollama

По умолчанию бот пытается использовать локальную Ollama:

ollama pull llama3.2:3b
ollama serve

Если Ollama недоступна или отключена, игра продолжит работать через fallback-тексты. Механика игры не зависит от LLM: код сам считает последствия, выбирает исходы и финалы, а LLM только формулирует художественный текст.
